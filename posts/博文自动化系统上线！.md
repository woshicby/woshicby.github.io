终于把博客系统升级了！以后写新博文不用再手动转JSON了，直接写Markdown就行，太方便了～

# ✨ 新功能介绍

之前每次写新博文都要：
1. 写Markdown文件
2. 打开convert-md-to-json.html页面
3. 上传文件转换
4. 复制JSON
5. 手动粘贴到posts.json
6. 提交到GitHub

一套流程下来真的很繁琐😅

现在只需要3步就能搞定！

## 📝 使用方法

### 第一步：写Markdown

在`posts/`目录下创建新的`.md`文件，格式像这样：

```markdown
---
    title: 博文标题
    date: 2026-02-20
    categories: [分类1, 分类2]
    tags: [标签1, 标签2]
    excerpt: 博文摘要
---

# 博文正文

这里是你的博文内容，支持完整的Markdown语法。
```

**注意点：**
- 必须有YAML frontmatter（就是`---`包围的那部分）
- `title`和`date`是必填的
- `categories`和`tags`用数组格式
- `excerpt`不写的话会自动从正文提取前200个字符

### 第二步：更新posts-list.json

在`JSON/posts-list.json`里加一条：

```json
{
  "file": "你的博文文件名.md",
  "id": 4
}
```

**重要：**
- `file`要和`posts/`目录下的文件名完全一样
- `id`要唯一，建议递增
- 加到数组末尾就行

### 第三步：提交GitHub

把新创建的`.md`文件和更新后的`posts-list.json`提交到GitHub，GitHub Pages会自动部署～

就这么简单！

# 🔍 系统怎么工作的？

1. 网页加载时先读`JSON/posts-list.json`
2. 根据文件列表逐个读取`posts/`目录下的`.md`文件
3. 解析Markdown的YAML frontmatter和正文内容
4. 自动提取标题、日期、分类、标签这些信息
5. 用markdown-it库把Markdown转成HTML渲染出来

# 💪 优势

- ✅ **不用手动转换**：直接写Markdown就行
- ✅ **版本控制友好**：Markdown文件好diff和review
- ✅ **自动解析**：系统自动提取元数据和摘要
- ✅ **向后兼容**：还支持原来的`posts.json`方式

# 🔄 向后兼容

如果`posts-list.json`不存在，系统会自动回退到读`JSON/posts.json`，所以旧的博文不会受影响～

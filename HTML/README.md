# HTML 目录说明文档

本目录包含网站的HTML页面文件，采用模块化设计，每个文件对应一个功能页面。

## 文件结构

```
HTML/
├── pace-calculator.html  # 配速计算器页面
├── sports.html           # 体育页面
├── study.html            # 学习页面
├── style-comparison.html # 样式比较器页面
├── tools.html            # 工具集页面
├── touken-forge.html     # 锻刀模拟器页面
├── video.html            # 视频播放器页面
└── README.md             # 目录说明文档
```

## 文件说明

### index.html（根目录）

**网站首页**，包含以下内容：
- 网站导航栏（支持响应式和主题切换）
- 个人介绍区域
- 项目展示区域
- 技能展示区域
- 联系信息
- 页脚
- 主题切换按钮

### pace-calculator.html

**配速计算器页面**，提供跑步配速计算功能：
- 跑步距离和时间输入表单
- 自动计算配速结果
- 可视化配速显示
- 结果分享功能
- 集成B站API支持
- 返回小工具页面按钮（样式引用自tools-common.css）

### sports.html

**体育页面**，展示体育相关内容：
- 体育新闻或文章列表
- 运动知识分享
- 训练计划或建议
- 支持深色/浅色主题切换

### study.html

**学习页面**，提供学习资料和笔记功能：
- 学习资料分类展示
- 笔记整理和管理
- 学习资源链接
- 响应式设计，适配各种设备

### style-comparison.html

**样式比较器页面**，提供不同样式方案的对比功能：
- 多视图样式对比
- 实时样式切换
- 样式参数调整
- 对比结果导出
- 与主题切换系统兼容

### tools.html

**工具集页面**，集合各类实用工具：
- 工具卡片式展示
- 工具分类筛选
- 工具使用说明和链接
- 统一的工具页面导航系统

### touken-forge.html

**刀剑乱舞锻刀模拟器页面**，模拟游戏中的锻刀系统：
- 锻刀界面和交互
- 材料选择和投入
- 结果展示和历史记录
- 本地存储用户数据
- 游戏化界面设计

### video.html

**视频播放器页面**，提供视频播放功能：
- 视频播放器界面
- 播放控制按钮
- 视频信息和说明
- 与B站API集成
- 响应式视频播放器

## 设计规范

### HTML结构

- 采用语义化HTML5标签（`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`等）
- 清晰的嵌套结构和缩进
- 适当的注释说明主要区块功能
- 遵循无障碍设计原则，提供适当的ARIA属性

### 通用组件

所有HTML页面共享以下通用组件：

- **导航栏**：包含网站标志和主导航链接，小屏幕设备显示汉堡菜单
- **页脚**：包含版权信息、联系方式等
- **响应式容器**：确保内容在不同屏幕尺寸下正常显示
- **主题切换按钮**：允许用户切换深色/浅色主题

### 外部资源引用

每个HTML页面都引入以下外部资源：

- **CSS文件**：
  - 首先引入`../CSS/common.css`作为基础样式
  - 然后引入页面特定的CSS文件（如`../CSS/index.css`）
  - 工具类页面额外引入`../CSS/tools-common.css`

- **JavaScript文件**：
  - 引入`../JS/navigation.js`实现响应式导航功能
  - 引入`../JS/theme-toggle.js`实现主题切换功能
  - 页面特定功能的JS文件（如计算器逻辑等）
  - 特定页面可能引入`../JS/bilibili-api.js`用于B站API集成

## 开发指南

1. **添加新页面**：
   - 在HTML目录下创建新的HTML文件
   - 复制或继承基础HTML结构，包括头部、导航、主内容和页脚
   - 引入必要的CSS和JS文件（common.css, navigation.js, theme-toggle.js）
   - 添加页面特定内容和功能
   - 确保实现主题切换支持

2. **修改现有页面**：
   - 遵循现有的HTML结构和命名规范
   - 确保语义化标签的正确使用
   - 保持注释的清晰和更新
   - 修改时考虑深色/浅色主题下的显示效果

3. **响应式设计**：
   - 所有页面都必须适配桌面、平板和移动设备
   - 优先使用CSS媒体查询实现响应式布局
   - 确保在小屏幕设备上导航功能和主题切换正常工作
   - 使用弹性布局（Flexbox）和网格布局（Grid）实现复杂布局

4. **主题切换支持**：
   - 确保页面内容使用CSS变量定义颜色，以支持主题切换
   - 特殊颜色或背景需要在深色模式下有对应的样式
   - 测试页面在两种主题下的可读性和视觉效果

## 注意事项

- 所有HTML文件必须遵循W3C标准
- 确保所有外部资源路径正确
- 页面内容应与对应的CSS和JS文件功能保持一致
- 响应式导航功能由`../JS/navigation.js`提供，确保在每个页面都正确引入
- 主题切换功能由`../JS/theme-toggle.js`提供，确保在每个页面都正确引入
- 工具页面间共享的组件（如返回按钮）使用`tools-common.css`中定义的样式
- 对于有特殊布局需求的页面，确保不破坏整体的响应式结构
